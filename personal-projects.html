Turtle Trading Agent

A system of autonomous trading agents that run the classic "turtle" trend-following strategy against real-time market data, with a transformer model in training to push decisions beyond fixed rules.

View on GitHub ↗
Agentic Pipelines Microservices Redis Queue Brokerage REST / WebSocket APIs Informer (Transformer)

Overview

What it does

Each ticker is watched by its own agent: it connects to a brokerage's open APIs to read real-time OHLCV (open/high/low/close/volume) and order-book data, then decides on its own when to act on entry and exit signals for a trend-following strategy.

Rather than a single monolithic script, the system is split into microservices so signal detection and order submission can run concurrently and autonomously across many tickers, communicating over a Redis queue for fast, low-overhead messaging between agents.

Background

The "turtle" strategy

Turtle trading is a well-known trend-following approach originally taught to a group of novice traders in the 1980s: it enters positions on price breakouts beyond a recent high/low range and manages risk with volatility-based position sizing and trailing stops. This agent implements that breakout logic as its core signal, evaluated continuously against live market data instead of end-of-day prices.

Architecture

System design

Data

Real-time OHLCV and order-book data streamed from a brokerage's open APIs, watched across multiple stocks in parallel.

Coordination

Independent microservices handle signal detection and order submission, communicating through a Redis queue to stay fast and decoupled.

Agentic forecasting

An Informer model — a transformer architecture built for long time-series data — is being trained on real-time data so the agents can reason and act on forecasts, not just fixed rules.

BerrySmart

An ESP32-based hydroponic system growing strawberries with zero manual intervention — starting as a small prototype, aimed at commercial smart-farm scale.

View on GitHub ↗
ESP32 I2C / ADC Sensors PWM Control WiFi + NTP Grafana Cloud (InfluxDB) REST API

Overview

What it does

Environmental sensors track light, temperature, humidity, pH, and nutrient concentration (TDS/EC) in the growing environment, while a water pump and fan are automatically controlled to keep everything in range — on a timer, a day/time schedule, or reactively to temperature — without anyone touching a dial.

Architecture

Hardware & software

Sensing

A BH1750 light sensor and SHT40 temp/humidity sensor over I2C, plus pH and TDS/EC sensors over ADC, continuously read the growing environment and nutrient solution.

Control

A PWM-driven water pump — on an auto timer plus day/time scheduling — and a temperature-triggered fan (both via MOSFET) keep conditions in range on their own.

Connectivity

WiFi and NTP-synced time feed live readings to Grafana Cloud over InfluxDB, alongside a local web UI and REST API for manual control and scheduling.
